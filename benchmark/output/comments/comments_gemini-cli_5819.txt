<note>**[Code Defect]** The logic for handling `toolConfig.tools` is inconsistent and potentially buggy. 

When a tool is provided as an object (e.g., a `BaseTool` instance), it is registered into `subagentToolRegistry`. However, if it is a `FunctionDeclaration`, it is ignored in the loop (lines 314-317) and not added to the registry. Later, in `runNonInteractive`, the code attempts to handle `FunctionDeclaration` by checking `typeof tool === 'object' && 'schema' in tool` (line 388). 

If a user passes a `FunctionDeclaration` in `toolConfig.tools`, it will not be present in `subagentToolRegistry` during the validation phase (lines 298-318), meaning it will skip the "requires confirmation" check entirely. This creates a security/safety bypass where unvalidated tools can be executed.</note>
