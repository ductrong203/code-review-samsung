<note>**[Performance]** The current implementation performs a redundant memory copy. 
`env->NewByteArray(size)` allocates a new buffer, then `env->CallLongMethod` is invoked, and finally `env->GetByteArrayRegion` copies the data from the Java-allocated buffer into the native `buffer` pointer. 

If the Java `read` method is intended to fill the buffer, you should consider using `env->GetPrimitiveArrayCritical` or `env->GetByteArrayElements` to access the Java array directly without an extra copy, or ensure the JNI overhead is minimized. Additionally, if `size` is large, `NewByteArray` might fail or cause significant GC pressure.</note>
