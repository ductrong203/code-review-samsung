<note>The `mp_rand_in_range32` function uses `mp_round_next_power_of_2(range) - 1` as a mask. If `range` is 0 (which is prevented by `mp_assert(min < max)`), this is fine. However, if `range` is a power of 2, the mask is `range - 1`. If `range` is not a power of 2, the mask covers the next power of 2. This is a standard rejection sampling approach, but it is susceptible to an infinite loop if `mp_rand_next` were to return values that always fail the `ret >= range` check, though statistically impossible with a good PRNG.</note>
