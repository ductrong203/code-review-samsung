<note>**[Maintainability and Readability]** The _serialize_dispatcher function uses a massive match-case block that is becoming a 'God function' for serialization. As more types are added, this will become increasingly difficult to maintain and test.

**[Code Defect]** The `case _` block attempts to handle numpy types using `hasattr(obj, 'dtype')`. If `obj` is a numpy scalar (e.g., `np.int64(1)`), it has a `dtype` attribute, but the logic `np.issubdtype(obj.dtype, np.object_)` calls `serialize(obj.item())`. If `obj.item()` is not serializable, this will cause infinite recursion or a crash.

**[Performance]** The serialize function uses recursion for nested structures. While correct, it lacks a depth limit, which could lead to RecursionError or stack overflow on deeply nested JSON-like objects.</note>
