<note>**[Performance]** The use of std::remove_if followed by erase on a std::vector of unique_ptrs results in O(n) complexity for every operation completion. While likely small in practice, this is an inefficient way to manage a collection of active operations compared to a std::unordered_set or std::list if the number of concurrent operations grows.

**[Maintainability and Readability]** The manual removal of an operation from the vector using std::remove_if inside OnOperationFinished adds unnecessary complexity to the task management logic. This logic is coupled with the task's own lifecycle management.</note>
