**[Security Vulnerability]** The `isProxyTrusted` method relies on a custom HTTP header `X-Forwarded-Trusted-Proxy` to determine if a proxy is trusted. If the application is not behind a strictly configured network or if the proxy does not strip this header from incoming client requests, an attacker can spoof this header to bypass security checks, leading to potential authentication bypass or unauthorized access to sensitive features.

**[Code Defect]** The implementation of isProxyTrusted() relies on a custom header 'X-Forwarded-Trusted-Proxy' without verifying the source IP address of the request. This allows any client to spoof the 'trusted' status by simply adding this header to their request, bypassing the security intended by proxy-trusted-addresses.

**[Performance]** The isProxyTrusted method performs a configuration lookup (Configuration.getOptionalKcValue) on every single HTTP request. Configuration lookups can involve map lookups or string parsing, which is inefficient in a hot path.</note>
