The environment variable is read and parsed on every request. While likely acceptable, it is inefficient and makes the code more verbose. It would be cleaner to read this configuration once during initialization or via a configuration object.

**[Performance]** Reading the environment variable inside the request handler function causes a syscall/lookup on every single chain info query, which is inefficient in a hot path.

**[Security Vulnerability]** The environment variable `LINERA_CHAIN_INFO_RECEIVED_LOG_MAX_ENTRIES` is parsed without validation. If an attacker or misconfiguration sets this to a very large value, it could lead to excessive memory allocation or denial of service (DoS) when reading the received log.</note>
