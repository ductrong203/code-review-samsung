<note>**[Maintainability and Readability]** The print_info function contains significant code duplication. The pattern of resetting a buffer, formatting a string, and calling TextOut is repeated three times with minor variations.

**[Code Defect]** The snprintf call uses 'sizeof(buf)-1' as the size argument. While this prevents buffer overflow, it is inconsistent with the other calls in the same file and technically reserves an extra byte unnecessarily. More importantly, if the string is exactly 256 characters, the last character will be truncated.

**[Performance]** Texture and Buffer objects are being re-created and initialized inside the render loop every frame. This causes unnecessary memory allocation and driver overhead.

The use of a fixed-size char buffer (256+1) for UI text is a potential maintenance risk if the strings being formatted grow in length.</note>
