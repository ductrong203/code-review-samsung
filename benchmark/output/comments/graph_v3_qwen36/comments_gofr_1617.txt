<note>**[Performance]** The `retryConnect` function in `client.go` contains an infinite `for` loop that spawns a new goroutine every time `Connect` fails. Since `Connect` is called during initialization and potentially elsewhere, this creates an unbounded number of goroutines if the NATS server is unreachable, leading to a resource leak (goroutine leak) and eventual OOM crash.

**[Code Defect]** The `retryConnect` function contains an infinite `for` loop with no exit condition other than a successful connection. If the NATS server is permanently unavailable, this goroutine will run forever, consuming resources and potentially causing memory leaks or goroutine leaks in the application.

**[Security Vulnerability]** The retryConnect function in pkg/gofr/datasource/pubsub/nats/client.go contains an infinite loop without a maximum retry limit or exponential backoff. If the NATS server is unreachable, this goroutine will continuously attempt reconnection, consuming CPU and network resources. This creates a Denial of Service (DoS) vector against the application's resources.</note>
