<note>**[Performance]** The `getTranslatedText` method performs multiple null checks and object access operations (`resp.getBody()`, `body.getCode()`, etc.) sequentially. While not O(n^2), this creates a deep call chain for a simple validation. More importantly, if any of these intermediate objects are null, it throws an `IllegalStateException` which is a runtime exception, potentially crashing the thread if not caught by the caller (though the caller catches `Exception` in `apply`, the stack trace overhead is significant). A flatter validation structure would be more performant.

**[Maintainability and Readability]** The getTranslatedText method throws IllegalStateException with generic messages like 'Failed to invoke alitranslate, caused response error.' These messages don't include the actual status code or error details from the API response, making it hard to diagnose issues. The method should include more specific error information.</note>
