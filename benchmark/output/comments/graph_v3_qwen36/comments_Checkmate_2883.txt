<note>**[Security Vulnerability]** Regular Expression Denial of Service (ReDoS) and Command Injection via unsanitized user input in regex. The code uses `new RegExp(expectedValue).test(payload)` where `expectedValue` comes from the monitor configuration (user-controlled). If an attacker provides a crafted string with backtracking (e.g., `(a+)+b`), it can cause ReDoS. Additionally, if `payload` is later used in shell commands or if the regex is used in a context where injection is possible, this is dangerous. At minimum, ReDoS is a critical availability risk.

**[Performance]** A new `RegExp` object is compiled on every call to `requestHttp` when `matchMethod === 'regex'`. If this function is called frequently (e.g., in a monitoring loop), this creates significant CPU overhead and garbage. The regex should be compiled once outside the function or cached.</note>
