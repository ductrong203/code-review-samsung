<note>**[Code Defect]** The SQL query for PersonRepository.delete uses `in $1` instead of `in ($1)`. This is invalid SQL syntax for passing an array parameter to an IN clause. It should be `in ($1)` to correctly handle the array of IDs.

**[Security Vulnerability]** SQL Injection vulnerability introduced in PersonRepository.delete query. The change from `in ($1)` to `in $1` removes the parameter binding for the array of IDs.
