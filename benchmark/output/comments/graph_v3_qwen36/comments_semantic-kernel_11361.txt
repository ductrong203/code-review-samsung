<note>**[Code Defect]** The code calls `.FirstAsync()` on the result of `agent.InvokeAsync()`. `InvokeAsync` returns an `IAsyncEnumerable<ChatMessageContent>`. If the agent returns an empty sequence (e.g., due to an error or empty response), `.FirstAsync()` will throw an `InvalidOperationException` (or `TaskCanceledException` depending on the provider) because there are no elements. This causes a crash instead of handling the empty case gracefully.

**[Performance]** Calling .FirstAsync() on the result of InvokeAsync() forces the enumeration of the entire async stream to find the first element. If the agent returns multiple messages or chunks, this discards the rest, which is inefficient. Additionally, .FirstAsync() throws if the sequence is empty, which is a potential runtime error rather than a graceful handling of no response.</note>
