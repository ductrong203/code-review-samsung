<note>**[Code Defect]** State mutation during render: `generateIframeSrc` is called directly in the component body (line 58) and reads `visibleAvatar` and `locale` state. This function is also used inside the template string for the iframe src. Calling a function that reads React state during the render phase causes the component to re-render immediately after rendering, leading to an infinite loop. The `iframeSrc` should be computed inside a `useMemo` or `useEffect` hook, or directly in the JSX/template using the state variables.

**[Performance]** The function `generateIframeSrc` is called on every render of the `EmbedModal` component (line 56). This causes string concatenation and URL construction on every single render, even when the state (`visibleAvatar`, `locale`) hasn't changed. This is an unnecessary recomputation that should be memoized.</note>

<note>**[Security Vulnerability]** The generateIframeSrc function constructs a URL using user-controlled state variables (visibleAvatar, locale) without sanitization. While the current implementation uses hardcoded keys, the locale value is directly interpolated into the query string. If the backend does not strictly validate the 'locale' parameter, this could lead to Open Redirect or XSS if the locale value is reflected in the response or used in a way that allows script injection. Additionally, the 'auth' parameter is passed directly from props without validation, which could be a security risk if not properly secured server-side.

**[Maintainability and Readability]** The function `generateIframeSrc` is defined inside the component body but outside any lifecycle hooks or memoization. This means it is recreated on every render, and more critically, `iframeSrc` is calculated synchronously during render (line 56). This causes the iframe source to update only on re-renders triggered by state changes, but the logic is tightly coupled with the render cycle. It should be wrapped in `useMemo` to ensure it only recalculates when dependencies change, improving performance and clarity.

**[Performance]** The `generateIframeSrc` function performs string concatenation and conditional checks. While small, doing this inside the component body without memoization means it runs on every render. If this component renders frequently or is part of a large list, this adds up.</note>
