<note>**[Security Vulnerability]** Race condition in concurrent map writes. The code uses a custom SyncMap to store file digests in parallel goroutines. However, the `filepath.Base(f)` extraction is performed inside the goroutine without synchronization relative to the map key generation. If multiple files share the same basename (e.g., 'model.bin' in different subdirectories), they will overwrite each other's entries in the map, leading to data integrity failures where the wrong digest is associated with a file name. This is a logic flaw that compromises data integrity in a security-sensitive context (blob creation).

**[Code Defect]** Inefficient map operations: `files.Store(filepath.Base(f), digest)` is called inside a goroutine. If `filepath.Base(f)` returns the same base name for multiple files, later stores will overwrite earlier ones, leading to data loss. Consider if this is intended behavior or if a different key strategy is needed.</note>

<note>Symlink traversal vulnerability. The code resolves symlinks using `filepath.EvalSymlinks` and then checks if the relative path is local using `filepath.IsLocal`. However, `filepath.IsLocal` only checks if the path starts with '..' or is absolute. It does not prevent traversal through symlinks that point outside the intended directory structure if the base path itself is a symlink. An attacker could create a symlink inside the model directory pointing to a sensitive file outside, and if the base path is not strictly validated, this could lead to reading arbitrary files.</note>
