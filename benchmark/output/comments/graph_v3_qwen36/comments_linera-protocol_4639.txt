<note>**[Maintainability and Readability]** Code duplication: The logic to read the environment variable and parse it into `max_entries` is duplicated in both `state.rs` and `client/mod.rs`. This violates DRY principles. If the parsing logic or default value changes, it must be updated in multiple places, increasing the risk of inconsistency.

**[Performance]** The environment variable `CHAIN_INFO_RECEIVED_LOG_MAX_ENTRIES_VAR` is read and parsed on every call to `prepare_chain_info_response`. This is an expensive operation performed frequently. It should be cached or read once at startup/configuration load time.

Missing error handling: The code silently falls back to the default value if the environment variable is invalid or missing. While this prevents crashes, it makes debugging configuration issues difficult. It is better to log a warning when the configuration is invalid so developers know why the default is being used.</note>
