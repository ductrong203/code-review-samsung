<note>**[Performance]** The `atLeastOneNameMatchesPattern` method uses `Arrays.stream(names).anyMatch(...)` which creates a new Stream object and an Iterator for every call. In a tight loop filtering many objects (like `loadTreeItems`), this generates significant garbage. A simple enhanced for-loop over the `names` array would be more efficient and avoid allocation.

**[Security Vulnerability]** The method `atLeastOneNameMatchesPattern` uses `Arrays.stream(names)` without validating if `names` is null. If a caller passes a null array, this will throw a NullPointerException. While not a direct security exploit, it can lead to Denial of Service (DoS) via application crash if untrusted input reaches this point.</note>
