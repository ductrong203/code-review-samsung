<note>**[Maintainability and Readability]** The lambda `is_always_monotonic_chain` is defined inside the function `matchesExactContinuousRange`. This creates a closure that captures `field` by reference. While functional, this inline definition makes the logic harder to read and test. It should be extracted to a private static helper method or a free function to improve separation of concerns and readability.

**[Performance]** The lambda `is_always_monotonic_chain` is defined inside the function but called multiple times within the loop over `rpn`. Each call iterates through the `chain` vector. If `rpn` is large and chains are long, this could be O(N*M) where N is number of RPN elements and M is average chain length. This is the core logic change, but the complexity is acceptable for the intended optimization. However, `func->getMonotonicityForRange` is called repeatedly for the same function and range. If this is expensive, it could be a bottleneck.</note>
