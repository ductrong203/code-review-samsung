<note>The 'content' field is declared as optional in the parser (using optionalConstructorArg), but the constructor argument passed to declareField is still constructorArg(). In Elasticsearch's ObjectParser, if a field is declared with constructorArg() but the parser marks it as optional, the parser will still expect the field to be present in the JSON (or it will fail to find it). To make the field truly optional during deserialization, the first argument to declareField must also be optionalConstructorArg(). Currently, the code changes the lambda behavior but not the constructor argument type, meaning the field remains effectively required or causes a parsing error if missing.</note>
