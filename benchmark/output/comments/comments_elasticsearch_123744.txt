<note>**[Code Defect]** The `isPathOnDefaultFilesystem` method returns `false` when the filesystem is not the default, which causes `checkFileRead` and `checkFileWrite` to return early (effectively bypassing all entitlement checks). 

If a path is on a non-default filesystem (e.g., a custom `FileSystem` implementation or a virtual filesystem), it should likely be subjected to stricter scrutiny or handled explicitly rather than being "trivially allowed" by returning early. Bypassing security checks for non-default filesystems creates a potential security vulnerability where unauthorized access to sensitive resources could occur if they are accessed via a non-default `FileSystem` provider.</note>
