<note>**[Code Defect]** Potential pointer arithmetic error in `lv_xml_split_str`.

The code uses `*src[0]`, which is equivalent to `**src`. If `src` is a pointer to a pointer (as indicated by the signature `char ** src`), `src++` increments the pointer to the pointer, not the character pointer itself. This will likely lead to an out-of-bounds memory access or incorrect logic when the caller expects the original `char *` to be updated.

If the intention is to skip delimiters in the string pointed to by `*src`, it should be:
`while(**src == delimiter) (*src)++;`</note>
