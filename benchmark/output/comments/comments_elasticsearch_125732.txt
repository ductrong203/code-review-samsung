<note>**[Maintainability]** The logic `ExceptionsHelper.status(e).getStatus() < 500` is used to determine whether to log at `DEBUG` or `WARN` level. While this aligns with `RestResponse`, it is fragile because `ExceptionsHelper.status(e)` can return `RestStatus.INTERNAL_SERVER_ERROR` (500) for many exceptions that are not necessarily "server errors" in the context of search execution (e.g., certain query parsing errors). 

Additionally, `ExceptionsHelper.isNodeOrShardUnavailableTypeException(e)` is a good check, but ensure that this logging doesn't become overly verbose in production if a cluster is experiencing widespread shard unavailability, as this will now trigger a `WARN` log for every failed shard search. Consider if a rate-limiting mechanism or a more specific exception filter is needed to prevent log flooding.</note>
