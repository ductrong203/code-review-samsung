<note>The TestEntitlementsRule uses a static AtomicBoolean 'active' to enforce test isolation. If a test suite fails or is interrupted in a way that prevents the 'finally' block from executing, 'active' remains true, causing all subsequent tests in the JVM to fail or potentially bypass entitlement checks if the logic is incorrectly assumed to be reset. This creates a risk of inconsistent security state across tests.</note>
