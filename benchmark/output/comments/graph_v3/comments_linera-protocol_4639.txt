<note>**[Code Defect]** The loop condition `received_entries < max_entries` is insufficient to detect the end of the log. If the total number of entries is exactly a multiple of `max_entries`, the loop will perform an extra request that returns an empty list, which is inefficient but acceptable, but if the server returns fewer than `max_entries` because it reached the end of the log, the loop terminates correctly. However, if the server returns exactly `max_entries` but there are more entries, the loop continues. The logic is correct, but the variable `max_entries` is parsed inside the loop, which is inefficient.

**[Performance]** The environment variable is parsed inside a loop in synchronize_received_certificates_from_validator. This redundant work should be hoisted outside the loop.</note>
