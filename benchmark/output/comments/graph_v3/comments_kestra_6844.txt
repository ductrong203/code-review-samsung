<note>**[Code Defect]** The error message construction uses `runContext.render(errorMessage)` which returns a `Property<String>`. Calling `.as(String.class)` on a `Property` object is incorrect; it should be called on the result of the render or the property should be evaluated properly. This will likely fail to render the actual string content of the error message.

**[Performance]** The error message construction performs a redundant render call inside the exception block. Since this only executes when a failure occurs, it is not a hot-path performance issue, but it is inefficient to re-render the property if it has already been evaluated or is static.</note>
