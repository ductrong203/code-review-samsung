<note>**[Code Defect]** The implementation of isProxyTrusted relies on a custom header 'X-Forwarded-Trusted-Proxy' to determine trust. This is insecure because an attacker can spoof this header to bypass the proxy trust check, effectively allowing any request to be treated as if it came from a trusted proxy.

**[Security Vulnerability]** The isProxyTrusted method relies on a user-controlled HTTP header 'X-Forwarded-Trusted-Proxy' to determine if a proxy is trusted. An attacker can spoof this header to bypass security checks that rely on isProxyTrusted(), such as X.509 certificate lookup, leading to potential authentication bypass.

**[Maintainability and Readability]** The isProxyTrusted implementation in QuarkusHttpRequest relies on a hardcoded header name 'X-Forwarded-Trusted-Proxy'. This magic string should be defined as a constant to improve maintainability and avoid potential typos across the codebase.</note>
