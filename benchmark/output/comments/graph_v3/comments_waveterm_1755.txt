In Go, this creates a new string object in every iteration, leading to O(n²) memory allocation behavior. Use strings.Builder instead.

If `HardQuote` does not correctly handle all shell-specific injection characters (like backticks or command substitution), an attacker controlling the environment variables could achieve command injection when the shell evaluates the exported variables.</note>
