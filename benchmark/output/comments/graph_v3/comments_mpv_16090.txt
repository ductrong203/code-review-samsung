<note>**[Code Defect]** The code calls mp_image_vflip(mpi) on the input image buffer before uploading it to the GPU. This modifies the image data in-place. If the same image buffer is used by other filters or components later in the pipeline, this will cause data corruption or unexpected behavior because the buffer is now permanently flipped.

**[Performance]** The call to mp_image_vflip(mpi) inside the upload loop performs a full CPU-based image transformation on every frame. This is an expensive memory-copy operation that should be avoided in the hot path. Since the GPU renderer supports vflip (as indicated by VO_CAP_VFLIP), this transformation should be handled via shader/texture coordinates or GPU-side matrix transformations instead of CPU-side pixel manipulation.</note>
