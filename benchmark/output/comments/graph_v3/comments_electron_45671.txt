<note>**[Code Defect]** The GetDevices method initiates an asynchronous Bluetooth adapter request but does not handle the case where the adapter might be unavailable or the request fails. If the adapter is not present, the code simply returns without ever calling GetDevices again or fetching the serial ports, leaving the UI in a permanently loading state.

**[Maintainability and Readability]** The GetDevices method uses a nested conditional structure to check for Bluetooth adapter availability. This increases cyclomatic complexity. Consider refactoring the adapter retrieval logic into a helper or using a more linear flow to improve readability.</note>
