<note>**[Maintainability and Readability]** Swallowing exceptions and returning null is an anti-pattern that makes debugging difficult for callers. It hides the root cause of the failure (e.g., network issues, auth errors). Consider throwing a custom unchecked exception or returning a Result/Optional object.

**[Security Vulnerability]** The catch block logs the exception message directly using logger.error. If the exception message contains sensitive information (e.g., API keys, internal paths, or PII), this could lead to sensitive data exposure in logs. Always sanitize or redact sensitive information before logging.</note>
