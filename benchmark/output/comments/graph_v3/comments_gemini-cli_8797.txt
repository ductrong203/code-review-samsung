<note>**[Code Defect]** The regex conversion logic for glob patterns is flawed. Replacing '/*' with '(/.*)?' is incorrect because it creates an optional group that matches empty strings or paths that don't start with a slash, potentially leading to incorrect extension enablement/disablement matches.

**[Security Vulnerability]** The globToRegex function uses a regex replacement that is vulnerable to ReDoS (Regular Expression Denial of Service). The pattern `(\/?)\*` replaced with `($1.*)?` can lead to catastrophic backtracking when processing deeply nested or crafted path strings, especially since it is applied to user-controlled input.

**[Maintainability and Readability]** The regex replacement logic `.replace(/(\/?)\*/g, '($1.*)?')` is complex and lacks comments. Regex-based path matching is notoriously difficult to debug; consider adding a comment explaining the transformation or using a dedicated glob-to-regex library.</note>
