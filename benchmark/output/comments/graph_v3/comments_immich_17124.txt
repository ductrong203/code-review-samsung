<note>**[Security Vulnerability]** The use of raw SQL template literals with user-provided input (ids) in the `getMetadataForIds` method creates a potential SQL injection vulnerability. While `ids` is passed to the `where` clause, the `sql<Date>` template literal uses string interpolation for the `AT TIME ZONE` expression. If this were to be extended or if the input handling changes, it could lead to injection. Always use parameterized queries for all dynamic SQL parts.

**[Performance]** The use of `AT TIME ZONE 'UTC'::text` inside the `min` and `max` aggregate functions forces a timezone conversion for every single asset row in the join before aggregation. This prevents the database from utilizing indexes on the `localDateTime` column effectively and significantly increases CPU overhead during the aggregation phase for large datasets.</note>
