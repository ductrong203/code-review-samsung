<note>The implementation of Write7BitEncodedInt uses a bitwise OR with ~0x7Fu (which is 0xFFFFFF80) to set the continuation bit. This is correct for unsigned integers, but the method accepts a signed 'int' and casts it to 'uint'. If a negative integer is passed, the cast to uint results in a very large value, which will be encoded incorrectly as a 5-byte sequence. While this might not lead to RCE, it causes data corruption in the AAD template, potentially leading to authentication bypass or integrity failures.</note>
