<note>**[Performance]** The create method performs a conditional check and object instantiation every time a highlight request is processed. While the overhead is minimal, this is a hot path in search execution. Consider caching the QueryMaxAnalyzedOffset instance if the input parameters are identical to avoid repeated object allocation.

**[Security Vulnerability]** The `create` method allows a negative value (specifically -1) to be passed through to the `QueryMaxAnalyzedOffset` constructor, which then uses the `indexMaxAnalyzedOffset` value. If `indexMaxAnalyzedOffset` is configured to a very large value or if the logic is misused, it could lead to resource exhaustion (DoS) by bypassing the intended per-query highlighting limits. While -1 is intended as a sentinel, ensure that the index-level setting is strictly validated to prevent excessive memory consumption during highlighting.</note>
