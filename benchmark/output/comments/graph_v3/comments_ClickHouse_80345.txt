<note>**[Performance]** The registration logic for array functions creates multiple large `FunctionDocumentation` objects and their associated `std::vector` and `std::string` members inside the `REGISTER_FUNCTION` block. Since this block is executed during server startup, creating these objects repeatedly for every function registration increases startup time and memory overhead. These should be defined as `static const` or `constexpr` to avoid heap allocations and object construction at runtime.

**[Maintainability and Readability]** The REGISTER_FUNCTION block has become a 'God function' containing massive amounts of boilerplate documentation code for multiple functions. This violates the Single Responsibility Principle and makes the registration logic hard to read. These documentation structures should be moved to separate static helper functions or dedicated metadata files.</note>
