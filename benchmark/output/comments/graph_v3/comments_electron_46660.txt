<note>**[Code Defect]** The `extension_description_overrides` vector and `extensions` vector in `ui::SelectFileDialog::FileTypeInfo` are expected to be parallel arrays. By moving the push_back of the description into the `else` block, the indices will no longer align when `has_all_files_wildcard` is true, causing the file dialog to display incorrect descriptions for the remaining filters.

**[Maintainability and Readability]** The logic for adding extension descriptions is now split across branches, creating an implicit dependency between the description list and the extensions list. If the 'all files' wildcard is present, the description is skipped, which may lead to index mismatches if the UI component expects a 1:1 mapping between descriptions and extension groups.</note>
