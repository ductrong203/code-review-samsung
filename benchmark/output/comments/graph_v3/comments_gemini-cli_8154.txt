<note>**[Security Vulnerability]** The `applyUpdates` function performs a recursive merge of user-provided settings into the existing configuration object. Because it uses `Object.getOwnPropertyNames(updates)` and directly assigns properties to the `result` object (which is a reference to the parsed JSON), it is vulnerable to Prototype Pollution. An attacker who can influence the `updates` object (e.g., via a malicious settings input) could inject properties like `__proto__`, `constructor`, or `prototype`, potentially leading to RCE or application-wide logic bypass.

**[Performance]** The `applyUpdates` function performs a deep recursive merge on every settings update. While functional, it uses `Object.getOwnPropertyNames` and manual recursion which can be inefficient for large settings objects. Given that settings files are typically small, this is acceptable, but it should be noted that this implementation does not handle circular references or complex prototype chains, which could lead to stack overflows if the input `updates` object is malformed.

**[Maintainability and Readability]** The `applyUpdates` function performs deep mutation on the `current` object passed as an argument. This side effect makes the function harder to reason about and debug, as it modifies the input object directly rather than returning a new, updated structure. It should be refactored to treat the input as immutable or explicitly document the mutation.</note>
