<note>**[Code Defect]** Using `fmt.Fscanln(os.Stdin, &line)` to read a JWT token is fragile. 

`Fscanln` stops at whitespace, which might truncate a JWT token if it contains certain characters or if the input stream is malformed. Additionally, it does not handle potential trailing newlines or carriage returns reliably in all terminal environments. It is safer to use `bufio.NewScanner(os.Stdin)` or `bufio.NewReader(os.Stdin).ReadString('\n')` to ensure the entire token line is captured correctly.</note>
