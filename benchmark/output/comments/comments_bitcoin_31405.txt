Ensure that `m_best_header` is always guaranteed to be the block with the most work that is not marked as invalid.</note>

<note>**[Code Defect]** The use of `Assume(state.IsInvalid())` inside `AcceptBlock` is risky. `Assume` is intended for conditions that should be impossible; however, `state.IsInvalid()` is a standard outcome of `CheckBlock` or `ContextualCheckBlock` when a block is rejected. If `state.IsInvalid()` returns `false` (e.g., a block is valid), `Assume` will trigger an assertion failure in debug builds, even though the code flow is perfectly valid. 

If the intention is to only call `InvalidBlockFound` when the block is actually invalid, use a standard `if (state.IsInvalid())` check instead of `Assume`.</note>
