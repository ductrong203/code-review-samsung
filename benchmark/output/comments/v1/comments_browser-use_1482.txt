<note>**[Maintainability]** The logic for constructing `chrome_launch_args` uses a `set` literal to remove duplicates. While this is clever, it is fragile because `set` does not guarantee order. Chrome command-line arguments are generally order-independent, but some flags (like those that override others) can behave unexpectedly if the order is non-deterministic. Furthermore, using a `set` prevents the use of arguments that might need to appear multiple times (though rare for Chrome).

Consider using a list and a helper function to filter duplicates while preserving order, or simply rely on the fact that `BrowserConfig` and `CHROME_ARGS` should be managed to avoid duplicates at the source.</note>
