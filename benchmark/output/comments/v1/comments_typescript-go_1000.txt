<note>**[Performance]** The current implementation of `GetDocumentDiagnostics` performs a sequential execution of multiple diagnostic providers (`GetSemanticDiagnostics`, `GetSuggestionDiagnostics`, `GetDeclarationDiagnostics`). 

In an LSP context, these operations are often I/O or CPU intensive. Since these calls are independent of each other, they should be executed concurrently using `errgroup` or `sync.WaitGroup` to reduce the latency of the diagnostic response, which directly impacts the user's perceived editor performance.</note>
