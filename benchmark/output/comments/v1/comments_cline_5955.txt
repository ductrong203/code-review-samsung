<note>**[Code Defect]** Potential memory leak / logic error with `bufferStuckTimer`.

In `flushBuffer`, the `bufferStuckTimer` is assigned a new `setTimeout` every time the function is called. If `flushBuffer` is called multiple times in rapid succession (e.g., due to high-frequency terminal output), the previous timer is not cleared before the new one is assigned. This leads to orphaned timers that will fire incorrectly.

**Fix:** Add `if (bufferStuckTimer) clearTimeout(bufferStuckTimer);` before assigning the new timer.</note>
