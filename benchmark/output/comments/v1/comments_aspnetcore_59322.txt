<note>**[Code Defect]** In `Write7BitEncodedInt`, the line `target[index++] = (byte)(uValue | ~0x7Fu);` is logically incorrect for 7-bit encoding. 

The standard way to set the continuation bit (the high bit) is `(byte)((uValue & 0x7F) | 0x80)`. Using `uValue | ~0x7Fu` performs a bitwise OR with a mask that has all bits set except the lowest 7. This will result in `0xFF` for any value where the high bits are set, effectively corrupting the data. 

Fix: `target[index++] = (byte)((uValue & 0x7F) | 0x80);`</note>

<note>**[Maintainability]** The test case ` "😀"[0] + "X"` is fragile. `😀` is a surrogate pair in C# (UTF-16). Accessing `[0]` returns the high surrogate, which is an invalid standalone character. While this correctly triggers an `EncoderFallbackException` (as intended for the test), it is better to use a known invalid sequence explicitly, such as `new string(new char[] { '\uD800' })`, to make the intent of the test clearer to other developers.</note>
