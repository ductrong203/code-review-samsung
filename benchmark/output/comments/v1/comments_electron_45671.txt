<note>**[Code Defect]** The use of `CHECK(adapter)` is dangerous here. 

`BluetoothAdapterFactory::Get()->GetAdapter` can return a null adapter in certain environments (e.g., if Bluetooth is unsupported or disabled at the system level). If this happens, the process will crash. You should handle the null case gracefully, perhaps by logging an error or updating the UI to reflect that Bluetooth is unavailable, rather than crashing the browser process.</note>

<note>**[Maintainability]** The `AdapterPoweredChanged` method triggers `GetDevices()` whenever the adapter is powered on. 

If the user has multiple Bluetooth devices or if the adapter toggles state rapidly, this could lead to excessive calls to `GetDevices()` and subsequent UI refreshes. It is recommended to debounce this call or ensure that the state has actually changed in a way that would affect the list of available serial ports before triggering a full refresh.</note>
