<note>**[Code Defect]** Potential Use-After-Free / Double Free.

The `ClearDataTask` class now inherits from `gin_helper::CleanedUpAtExit`. If the `CleanedUpAtExit` mechanism triggers the destructor (e.g., during Node environment shutdown), it will delete the object. However, `OnTaskFinished` also calls `delete this`. 

If `OnTaskFinished` is called while the object is being cleaned up by the `CleanedUpAtExit` registry, or if the registry attempts to clean up an object that has already been deleted by `OnTaskFinished`, this will lead to a double-free or use-after-free. You should ensure that `CleanedUpAtExit` is properly unregistered or that the destruction logic is idempotent.</note>

<note>**[Maintainability]** Manual memory management (`new`).

The code transitions from `std::shared_ptr` to raw `new` for `ClearDataTask`. While you are managing the lifetime via `CleanedUpAtExit` and `delete this`, this pattern is error-prone in modern C++.
