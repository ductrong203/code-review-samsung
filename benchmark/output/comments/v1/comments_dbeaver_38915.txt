If `super.setDefaultCatalog` fails (e.g., due to a connection issue or invalid catalog), the UI will incorrectly assume the operation succeeded because `activeCatalogName` might be updated or the caller will not be notified of the failure.

<note>**[Performance]** Executing a metadata query (`SELECT CURRENT_CATALOG()`) inside `refreshDefaults` can cause performance degradation if `refreshDefaults` is called frequently.

While `refreshDefaults` is intended to sync state, performing a network round-trip for every refresh might be heavy. Additionally, if the session is not yet fully established or the connection is unstable, this query could hang or fail.

**Fix:** Ensure that this logic is only executed when necessary, or consider caching the result with a short TTL if the catalog is not expected to change frequently within the same session.</note>
