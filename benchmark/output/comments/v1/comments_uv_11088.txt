<note>**[Performance]** The current implementation performs multiple allocations per URL path segment. 

`url.path_segments().unwrap()` returns an iterator, and for each segment, you are calling `urlencoding::decode`, `into_owned()`, and then `collect::<Vec<_>>()`. This creates a new `String` for every segment and a `Vec` to hold them, even if no percent-encoding was present in the original path.

Since you already check for the presence of `%` using `memchr`, you should only perform the decoding logic if necessary. Furthermore, consider using a `String` buffer and `write!` or `push_str` to reconstruct the path in a single allocation rather than collecting into a `Vec` and then extending the `path_segments_mut`.</note>
