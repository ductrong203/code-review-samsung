<note>**[Performance]** The `flattenAndArchive` method performs a sequential `await` inside a `for` loop when archiving workflows. If a folder contains many workflows, this will result in N sequential database/service calls, significantly increasing the latency of the folder deletion operation.

Consider using `Promise.all` to execute these archive operations concurrently.</note>

<note>**[Performance]** The `getAllWorkflowIdsInHierarchy` method uses `query.getMany()` to fetch full `WorkflowEntity` objects just to extract their IDs. This is inefficient as it loads unnecessary columns into memory.

Use `query.select('workflow.id').getMany()` or `query.getRawMany()` to fetch only the required ID column, reducing memory usage and database load.</note>
