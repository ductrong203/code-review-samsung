<note>**[Performance]** Unnecessary Locking.

The logic inside `prepareSetProcessingRequests` uses `std::unique_lock` with `try_lock()` to handle potential contention between multiple tables on the same server. However, `metadata_ref_count` is an `std::atomic`. If `metadata_ref_count` is 1, the lock is skipped entirely. If a table is destroyed but the metadata remains in the factory, the ref count might stay high, causing unnecessary lock contention for single-table setups.</note>
