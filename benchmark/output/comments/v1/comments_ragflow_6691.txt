<note>**[Performance/Maintainability]** Using `lambda` inside a loop for `nursery.start_soon` can lead to closure capture issues (late binding). 

While `trio.to_thread.run_sync` is used correctly here to offload blocking I/O, the `lambda` captures the loop variables `from_node` and `to_node` by reference. If the loop continues before the thread starts, all tasks might execute with the values from the final iteration. It is safer to use `functools.partial` or define a helper function to capture the arguments immediately.</note>
