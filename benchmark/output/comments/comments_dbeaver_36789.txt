<note>**[Performance]** Potential SQL performance issue.

The query joins `M_CS_TABLES` and `TABLE_PARTITIONS` using `mcs.PART_ID = tp.PART_ID`. In SAP HANA, `M_CS_TABLES` contains one row per partition. If the intention is to fetch partitions for a specific table, joining on `PART_ID` is correct, but ensure that `TABLE_NAME` and `SCHEMA_NAME` are indexed in both tables to avoid a full table scan on `M_CS_TABLES`. Additionally, verify if `TABLE_PARTITIONS` is sufficient on its own, as it usually contains the partition metadata directly.</note>
