<note>**[Security Vulnerability]** The `isProxyTrusted()` implementation relies on a custom header `X-Forwarded-Trusted-Proxy` to override the trust decision. 

If this header is not stripped by the edge proxy, a malicious client can inject `X-Forwarded-Trusted-Proxy: true` to bypass the `proxy-trusted-addresses` check. Unless the underlying infrastructure (e.g., Quarkus/Netty) is explicitly configured to strip this header from incoming requests, this introduces a significant security risk where an untrusted client can spoof a trusted proxy status.</note>
